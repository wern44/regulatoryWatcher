How these processes work together

1

CSSF Discovery scrapes the CSSF website for new circulars, laws, and regulations and adds them to the Catalog. Run this first so the Pipeline has a complete catalog to match against.

2

Pipeline Run checks RSS feeds and SPARQL endpoints for new publications, matches them to catalog entries, and creates Inbox events. This is your main update monitor.

3

Full Reconciliation walks every page of the CSSF site and retires regulations no longer listed. Run weekly to keep the catalog clean.

4

Catalog Refresh & Analysis uses the LLM to classify regulations (ICT/DORA, entity applicability) and discover missing ones. Run monthly — it is LLM-intensive.

Processes never run simultaneously — each waits for the previous one to finish. If a process is already running when another is scheduled, the tick is skipped.
